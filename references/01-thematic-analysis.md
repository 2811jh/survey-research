# 主题分析法（Thematic Analysis）

合成定性研究的核心方法，适用于访谈、开放题等定性数据的系统化处理。

## 六步流程

1. **熟悉数据（Familiarization）**
   - 通读全部数据，先建立整体感知，再进行任何编码
   - 目标：对数据的整体面貌有直觉性理解，避免过早聚焦细节

2. **初始编码（Initial Coding）**
   - 系统地梳理数据，为每个观察、引用或数据点打上描述性标签
   - 宁可多标也不要少标——合并标签比拆分容易
   - 每个编码应该描述性地说明"这段内容在说什么"

3. **主题形成（Theme Development）**
   - 将相关编码归入候选主题
   - 一个主题应捕捉数据中与研究问题相关的重要内容
   - 主题不等于编码本身，它是更高层次的概括

4. **主题审查（Theme Review）**
   检查每个主题是否满足以下标准：
   - 是否有足够的数据证据支撑？
   - 主题之间是否相互独立、不重叠？
   - 所有主题放在一起是否讲述了一个连贯的故事？

5. **主题精炼（Theme Refinement）**
   - 清晰地定义和命名每个主题
   - 为每个主题写 1~2 句话的描述，说明它捕捉了什么
   - 确保命名准确反映主题内容，避免模糊或宽泛的命名

6. **输出报告（Report）**
   - 将各主题作为核心发现，附上支撑证据写成报告
   - 每个主题应有代表性引用或观察来佐证

## 质量检查

- **充分性**：每个主题至少有 3 条以上独立证据支撑
- **独立性**：主题之间不应大量重叠
- **穷尽性**：重要的数据模式都应被某个主题覆盖
- **相关性**：每个主题都应回应原始研究问题

## 在报告写作中的应用

- 将主题作为报告的章节或小节组织框架
- 每个主题配 2~3 条代表性用户原声
- 说明该主题在数据中的出现频率（如"12/20 位受访者提到"）
- 阐释主题的含义及其对产品决策的影响
