# 亲和力映射法（Affinity Mapping）

一种将大量分散观察分组归类的协作方法，常用于将访谈数据、开放题回答整理成有意义的结构。

## 五步流程

1. **捕获观察（Capture Observations）**
   - 将每个独立的观察、引用或数据点写成单独的条目
   - 一条记录只记录一个观察，不要合并多个洞察
   - 保留原始语言，不要过早解读

2. **聚类分组（Cluster）**
   - 根据相似性将相关条目归组
   - **不要预先定义分类**，让分类从数据中自然涌现
   - 初始分组不必完美，可以随时调整

3. **标注分组（Label Clusters）**
   - 给每个分组起一个描述性名称，捕捉其共同线索
   - 名称应该是对内容的概括，而非某一条具体观察
   - 好的命名：简洁、准确、有区分度

4. **组织分组（Organize Clusters）**
   - 如果发现更高层次的模式，将相关分组整理成更高层级的组合
   - 形成二级或三级结构，揭示数据中的层次关系

5. **识别主题（Identify Themes）**
   - 各分组及其关系最终揭示出关键主题
   - 主题应能回答研究问题

## 操作要点

| 要点 | 说明 |
|------|------|
| 单一性 | 每条记录只包含一个观察，不合并多个洞察 |
| 灵活性 | 可以自由移动条目，第一次分组往往不是最佳结果 |
| 拆分原则 | 如果某个分组过大，很可能包含多个子主题，应拆分 |
| 保留异常 | 异常点（outlier）有价值，不要强行归入某个分组 |
| 过程价值 | 分组的过程本身与输出结果同样重要，有助于建立共同理解 |

## 与文本分析的对接

在 survey-research 的文本分析流程中，亲和力映射法对应以下操作：
- **捕获观察** → 从 `text_extract.py` 提取的用户原文列表
- **聚类分组** → AI 将原文归入不同维度
- **标注分组** → 为每个维度命名（4~8 个维度）
- **识别主题** → 输出各维度的核心发现

## 常见错误

- 分组太宽泛：名称模糊，一个分组包含了不相关的内容
- 过度预设：先确定分组再往里塞数据，违背"让数据说话"的原则
- 强行穷尽：把不明确的条目随意归入某个组，导致分组不准确
- 分组过多：超过 8 个主维度会降低可读性和可操作性
