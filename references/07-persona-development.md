# 用户画像构建（Persona Development）

指导如何从研究数据中提炼出有据可查、对产品决策真正有用的用户画像。

## 一、核心原则

> **用户画像应从研究数据中涌现，而非凭空想象。**

一个好的用户画像必须满足：
- 能够在原始数据中找到对应的行为聚类
- 能够区分出不同画像在产品决策上的不同含义
- 能够用定量数据估算出该画像的用户规模

---

## 二、构建流程

### 步骤 1：识别行为模式
在参与者之间寻找**相似行为、目标和情境的聚类**，而非人口统计特征的相似。

重点关注：
- 使用频率和深度
- 主要使用目的（他们用这个产品解决什么问题）
- 在哪些环节遇到困难
- 对产品的期待和评价倾向

### 步骤 2：定义区分变量
确定哪些维度能够真正区分不同的用户聚类：

常用区分维度（根据实际数据调整）：
- 付费水平 / 消费频率
- 使用深度（轻度/重度）
- 主要使用场景或目的
- 技术熟练程度
- 新老用户身份

### 步骤 3：创建画像档案
每个画像包含以下内容：

| 字段 | 说明 |
|------|------|
| 名称 + 一句话描述 | 便于团队记忆和引用 |
| 关键行为与目标 | 他们在产品中做什么、想达成什么 |
| 痛点与未满足需求 | 现有产品没能解决好的地方 |
| 情境 | 职业角色、使用场景、配套工具 |
| 代表性引用 | 2~3 条能代表该画像视角的用户原声 |

### 步骤 4：用数据验证规模
- 能否通过基础统计或交叉分析估算每个画像的用户占比？
- 如果某个画像无法找到对应的数据量，需要重新审视其合理性

---

## 三、画像模板

```
[画像名称] — [一句话描述]

【他们是谁】
- 职位 / 用户类型 / 经验水平
- 如何开始使用本产品

【他们想实现什么】
- 主要目标和核心使用场景
- 如何衡量自己的成功

【他们如何使用产品】
- 使用频率和深度
- 常用功能和核心路径
- 与本产品配合使用的其他工具

【主要痛点（Top 3）】
- 痛点一
- 痛点二
- 痛点三
- 现有变通方法

【他们看重什么】
- 在解决方案中最重要的因素
- 什么会让他们流失或转向竞品

【代表性原声】
- "用户原话 1"
- "用户原话 2"
- "用户原话 3"
```

---

## 四、常见错误（需主动规避）

| 错误类型 | 问题 | 正确做法 |
|----------|------|----------|
| 人口统计型画像 | 用年龄/性别/地域定义，而非行为定义 | 行为和使用场景比人口统计更能预测产品需求 |
| 画像过多 | 超过 5 个画像，难以在实际决策中使用 | 3~5 个是最佳数量 |
| 虚构画像 | 基于团队假设创建，没有数据支撑 | 每个画像必须能在原始数据中找到对应证据 |
| 静态画像 | 创建后从不更新 | 随产品和用户群演变定期更新 |
| 无行动意义 | 画像无法影响任何产品决策 | 每个画像应有"产品含义"：这个画像要求产品在哪些方面做什么 |

---

## 五、在本 skill 中的应用

如果分析数据揭示出明显的用户分群（如交叉分析发现不同付费等级用户的回答存在系统性差异），可在综合报告的"用户洞察"章节中基于数据描述主要用户群体特征，但无需创建完整正式的画像档案——除非用户明确要求。
