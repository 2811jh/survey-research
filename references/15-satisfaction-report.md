# 满意度专项报告工作流程

> 📌 **何时读取本文档**：用户要求重点分析满意度、NPS、产品健康度、满意度变化时，由 SKILL.md 阶段 5 指引跳转至此。

## 触发关键词

满意度、NPS、净推荐值、满意度变化、产品健康度、满意度周报、满意度分析

## 一、数据准备

完成阶段 1（数据加载与分类）后，从列分类结果中识别以下关键题目：

### 1.1 识别核心指标题

| 题目类型 | 识别方法 | 示例 |
|---------|---------|------|
| **整体满意度** | 列名含"整体满意"/"总体满意"/"overall satisfaction"，且为 1-5 或 1-10 量表 | Q1.请问您对《xxx》的整体满意度？ |
| **NPS 推荐度** | 列名含"推荐"/"recommend"/"NPS"，且为 0-10 量表 | Q2.您有多大可能向朋友推荐本游戏？ |
| **细分维度满意度** | 矩阵量表题中子项含"满意"或属于同一满意度矩阵 | Q3.画面:满意度、Q3.音效:满意度... |

### 1.2 识别原因题

**路径 A（选项题）**：列名含"满意原因"/"不满原因"/"喜欢的地方"/"不喜欢的地方"等，为单选或多选题。
**路径 B（文本题）**：列名含"建议"/"原因"/"为什么"等开放题。

如果两种题型都存在，**优先使用选项题**（数据更结构化），文本题作为补充原声。
如果都不存在，跳过原因分析部分，在报告中注明"本次问卷未设置原因采集题"。

### 1.3 向用户确认

自动识别完成后，用一段文字告知用户识别结果，例如：
> 我识别到以下满意度相关题目：
> - 整体满意度：Q1.请问您对《xxx》的整体满意度？（1-5 分）
> - NPS 推荐度：Q2.您有多大可能向朋友推荐？（0-10 分）
> - 细分维度满意度：Q3 矩阵（共 8 个子维度）
> - 不满原因：Q5.您不满意的原因是？（多选题）
> - 满意原因：无专门题目，将从文本题 Q10 中归纳
>
> 确认无误我就开始分析。

## 二、指标计算

### 2.1 整体满意度

```python
# 计算逻辑
mean_score = df["满意度列"].mean()        # 均值
满分 = 该题量表最大值（5 或 10）
满意率 = (df["满意度列"] >= 4).mean()     # 5分制时 4+5 为满意
不满率 = (df["满意度列"] <= 2).mean()     # 5分制时 1+2 为不满
```

**量表适配规则**：
| 量表 | 满意 | 中性 | 不满 |
|------|------|------|------|
| 1-5 分 | 4-5 | 3 | 1-2 |
| 1-7 分 | 6-7 | 4-5 | 1-3 |
| 1-10 分 | 8-10 | 6-7 | 1-5 |

### 2.2 NPS 计算

```python
# NPS = 推荐者占比 - 贬损者占比（0-10 量表）
推荐者 = (df["NPS列"] >= 9).sum()     # 9-10 分
被动者 = ((df["NPS列"] >= 7) & (df["NPS列"] <= 8)).sum()  # 7-8 分
贬损者 = (df["NPS列"] <= 6).sum()     # 0-6 分
total = 推荐者 + 被动者 + 贬损者

NPS = (推荐者 / total - 贬损者 / total) * 100  # 结果为百分比
推荐者占比 = 推荐者 / total * 100
被动者占比 = 被动者 / total * 100
贬损者占比 = 贬损者 / total * 100
```

### 2.3 细分维度满意度

对矩阵量表中每个子维度：
- 计算均值
- 计算满意率（Top2）和不满率（Bottom2）
- 按均值从高到低排序

### 2.4 满意/不满原因统计

**路径 A（选项题）**：
- 统计各选项的选择频率和占比
- 满意原因取 Top3，不满原因取 Top10
- 如有对应的文本题，从中抽取每个原因的代表性原声（1-2 条）

**路径 B（文本题）**：
- 按阶段 4 文本分析流程处理（读取 `references/13-text-analysis-workflow.md`）
- 将归纳出的维度分为"正向"和"负向"两组
- 正向维度即满意原因（Top3），负向维度即不满原因（Top10）

## 三、预警规则

以下情况触发预警标记：

| 预警条件 | 预警级别 | 显示样式 |
|---------|---------|---------|
| 整体满意度 < 3.5（5 分制）或 < 7.0（10 分制） | 🔴 红色预警 | ⚠️ 整体满意度 X.X，低于健康线 |
| NPS < -20% | 🔴 红色预警 | ⚠️ NPS X%，贬损者占比过高 |
| NPS 在 -20% ~ 0% | 🟡 黄色预警 | ⚠️ NPS X%，需关注 |
| 某细分维度得分 < 3.0（5 分制） | 🔴 红色预警 | 🔻 维度名 X.X，重点关注 |
| 某细分维度不满率 > 30% | 🟡 黄色预警 | ⚠️ 维度名 不满率 X%，需关注 |
